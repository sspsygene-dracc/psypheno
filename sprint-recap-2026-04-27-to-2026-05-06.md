## 2. Effect-size visualizations (Mullen, Pollard, Tom)

- **Per-dataset volcano plot** on the gene-search page, FDR-based dot coloring (Up / Down / NS by FDR ≤ 0.05, falling back to p ≤ 0.05 when no FDR column), with the queried gene's rows overlaid as amber outlined circles. Tooltip labels include units and original column names; background is dimmed; dots are sized for legibility ([#71](https://github.com/sspsygene-dracc/psypheno/issues/71), [#107](https://github.com/sspsygene-dracc/psypheno/issues/107), [#108](https://github.com/sspsygene-dracc/psypheno/issues/108), [#109](https://github.com/sspsygene-dracc/psypheno/issues/109)). The first cut also shipped a histogram tab and a cross-table effect-size bar chart, but both were removed per your 2026-04-28 review — the histogram was space-hungry and the cross-table bar mixed wildly different studies on one axis.
- **Volcano background scoped to the queried perturbation:** when a perturbed gene is set and the table has a perturbed link direction, the red/blue background dots are now sampled from rows of *that perturbation experiment* rather than the whole table; sampling is deterministic per DB (top 100 by p-value plus a hash-sorted bulk sample, deduped). Target-only queries don't restrict the background (it would shrink to 1–2 rows) ([#152](https://github.com/sspsygene-dracc/psypheno/issues/152)).
- **Volcano falls back to FDR for one table without raw p-values:** `perturb_fish_astro` ships `lfc` + `qVal` only, so `/api/effect-distribution` used to 404 on it. It now uses `fdr_column` as the y-axis source when `pvalue_column` is missing and reports the actually-used column to the frontend, which renders `-log10(qval)`.
- **Signed effect sizes** are now first-class: `effect_column` declared per dataset, surfaced everywhere alongside p-values ([#70](https://github.com/sspsygene-dracc/psypheno/issues/70)).
- **Volcano lazy-loads** so gene-search results render fast even on slow connections.

## 8. Statistical rigor (#148)

A diagnostic audit revealed that several datasets' raw p-value distributions are not uniform-under-null, which biases the consortium-wide meta-analysis. Concretely:

- **Dropped Stouffer** as a default combiner — it's the most distortion-sensitive when p-values are right-censored.
- **Switched to Harmonic Mean P (HMP)** as the default combiner; **Cauchy Combination Test (CCT)** kept as the secondary cross-check. Both are robust to dependence and to the kind of right-censoring most supplementary tables have.
- **U[0,1] caveat documented** in the Methods page so users understand why the combined p-value is what it is.
- All of this is written up in `docs/fdr_meta_analysis_report.md` for future readers.

## 9. Performance and infrastructure

- **R meta-analysis cache** — meta-analysis results are now cached across `load-db` runs, dropping a slow 60×Rscript step to a near-instant cache hit on incremental rebuilds ([#162](https://github.com/sspsygene-dracc/psypheno/issues/162)). Cache hit path itself sped up further by streaming-hashing inputs and dropping the mpmath dependency.
- **Browser caches the read APIs** so navigating around the site is snappier on flaky networks.
- **`load-db --test` mode** restricts each dataset to a bundled top-genes fixture, capping each unique combination to 200 rows — exercises the full pipeline end-to-end in ~30s instead of 2–3 min ([#130](https://github.com/sspsygene-dracc/psypheno/issues/130)).
- **`sspsygene deploy --preprocess` and `--instances`** for selective deploys (e.g. dev only); per-instance restart and per-instance test gating ([#159](https://github.com/sspsygene-dracc/psypheno/issues/159) follow-up).
- **Test suite buildout** (no CI; single `scripts/test.sh` entry point):
  - Python backend tests: ~118 cases over the loader and preprocessing library ([#110](https://github.com/sspsygene-dracc/psypheno/issues/110)).
  - Frontend Vitest unit tests for the lib + API layer ([#111](https://github.com/sspsygene-dracc/psypheno/issues/111)).
  - Playwright end-to-end suite covering the core user flows — expanded from 7 to 88 tests over the sprint ([#112](https://github.com/sspsygene-dracc/psypheno/issues/112)).
  - Data-correspondence tests: row accounting + value spot-check against the loaded DB ([#113](https://github.com/sspsygene-dracc/psypheno/issues/113)).
  - All four are runnable from one script (`scripts/test.sh`) and gated into deploys via `sspsygene deploy --run-tests`.
- **Security review** of the sprint's changes ([#157](https://github.com/sspsygene-dracc/psypheno/issues/157)) — Next.js bumped, download-path hardened, URL validation tightened.

## 10. Wrangler-facing changes (more in the follow-up)

These primarily affect William, Brian, and Brittney; the follow-up email will go through the workflow changes in detail. In short:

- **Shared `processing.preprocessing` library** ([#149](https://github.com/sspsygene-dracc/psypheno/issues/149)): `preprocess.py` is now a `Pipeline` of tracked steps — `read_csv → clean_gene → dropna → filter_rows → write_csv`. Every step records what it did into a sibling `<output>.preprocessing.yaml` file ([#150](https://github.com/sspsygene-dracc/psypheno/issues/150), [#158](https://github.com/sspsygene-dracc/psypheno/issues/158)).
- **All ten in-tree datasets migrated** to the new pipeline form.
- **Gene-name cleanup architecture** revamped (Tiers A–D, [#120](https://github.com/sspsygene-dracc/psypheno/issues/120)–[#125](https://github.com/sspsygene-dracc/psypheno/issues/125), [#139](https://github.com/sspsygene-dracc/psypheno/issues/139)): Excel-mangled rescue (`9-Sep → SEPTIN9`), `make.unique` strip, ENSG → symbol at preprocess time (no runtime resolver), HGNC ID resolution, GENCODE clone resolution. Genuine-unknown warning count dropped from ~6,651 to 75 across the consortium.
- **Retired YAML knobs** `ignore_missing` / `replace` / `to_upper` / `drop_values` / `drop_patterns` — the schema rejects them now, with error messages pointing at the new home (`non_resolving.control_values` / `record_values` / `record_patterns` / `manual_aliases` in preprocess).
- **Controls** (`NonTarget1`, `GFP`, `SafeTarget`, …) are now first-class entries in `central_gene` with `kind='control'`, searchable via the autocomplete and via the `control` keyword, but excluded from per-gene aggregates ([#161](https://github.com/sspsygene-dracc/psypheno/issues/161)).
- **`docs/wrangler_gene_cleanup.md`** updated with the new vocabulary, open decisions still owed by wranglers (mostly successor-symbol triage for retired HGNC entries), and a verification recipe.
